# Telegram-бот для портфеля облигаций и акций

Бот на **aiogram 3.x**:
1. При первом `/start` регистрирует пользователя (SQLite).
2. Пользователь добавляет бумаги в портфель: тип (облигация/акция), тикер MOEX, количество.
3. Каждые `CHECK_INTERVAL_HOURS` часов (по умолчанию 6) бот сам присылает по каждой бумаге:
   - текущую цену и дневное изменение (MOEX ISS API),
   - 1–2 свежих новости по эмитенту,
   - информационный сигнал «держать / докупить / зафиксировать прибыль» на основе изменения цены за неделю.
4. Есть кнопка «📰 Сводка сейчас» — получить сводку вне расписания.

⚠️ Сигналы — это простая эвристика по цене, **не индивидуальная финансовая рекомендация**. В каждом сообщении есть дисклеймер.

## Структура проекта

```
telegram_bond_bot/
├── main.py                 # точка входа
├── config.py                # чтение .env
├── database/db.py           # SQLite: users, portfolio
├── handlers/                # /start, добавление/удаление бумаг, сводка
├── keyboards/                # reply/inline клавиатуры
├── services/
│   ├── moex.py               # цены и доходности с MOEX ISS (публичный API, без ключа)
│   ├── news.py               # новости: NewsAPI (по ключу) или Google News RSS (бесплатно)
│   ├── analysis.py           # эвристика рекомендаций
│   └── scheduler.py          # APScheduler: рассылка раз в N часов
├── states/forms.py           # FSM для добавления бумаги
├── tests/                    # offline-тесты БД и эвристики
├── requirements.txt
├── .env.example
└── Dockerfile
```

## Установка и запуск

1. Получите токен бота у [@BotFather](https://t.me/BotFather) (`/newbot`).
2. Скопируйте `.env.example` в `.env` и укажите `BOT_TOKEN`:
   ```
   cp .env.example .env
   ```
   `NEWS_API_KEY` необязателен — без него новости будут браться через бесплатный Google News RSS. Ключ NewsAPI (бесплатный тариф) можно получить на https://newsapi.org — даёт более релевантную выдачу.
3. Установите зависимости:
   ```
   pip install -r requirements.txt
   ```
4. Запустите бота:
   ```
   python main.py
   ```
   или через Docker:
   ```
   docker build -t bond-bot .
   docker run --env-file .env -v $(pwd)/data:/app/data bond-bot
   ```

## Как пользоваться

- `/start` — регистрация.
- «➕ Добавить бумагу» → выбрать тип → ввести тикер MOEX (например `SBER`, `GAZP` — акции; `SU26238RMFS4`, `RU000A1035V6` — облигации) → указать количество.
- «📊 Мой портфель» — список бумаг.
- «🗑 Удалить бумагу» — удалить одну из бумаг.
- «📰 Сводка сейчас» — получить сводку немедленно, не дожидаясь расписания.
- Автоматическая рассылка — каждые `CHECK_INTERVAL_HOURS` часов всем, у кого портфель не пуст.

## Что реально проверено в этой среде разработки

Песочница, в которой собирался проект, не имеет доступа к `api.telegram.org` и `iss.moex.com` (сеть ограничена служебными доменами вроде PyPI/GitHub), поэтому "живой" прогон бота с реальным Telegram и биржевыми данными здесь невозможен. Проверено то, что можно проверить без внешней сети:

- ✅ Установка всех зависимостей из `requirements.txt`.
- ✅ Синтаксическая корректность всех модулей (`py_compile`).
- ✅ Сборка приложения целиком: создание `Bot`, `Dispatcher`, регистрация всех роутеров, инициализация планировщика — без ошибок импорта/связывания.
- ✅ `tests/test_database.py` — регистрация пользователя (включая защиту от дублей), добавление/чтение/удаление бумаг, группировка портфелей по пользователям — на реальном SQLite через `aiosqlite`.
- ✅ `tests/test_analysis.py` — вся матрица эвристики рекомендаций (рост/падение/нейтрально/нет данных, акции и облигации).

Что стоит проверить самостоятельно при первом запуске с реальным токеном и интернетом:
- корректность конкретных тикеров MOEX (структура ответа ISS стабильна, но для облигаций поле `YIELD` заполняется не всегда — тогда бот покажет сигнал без доходности);
- лимиты NewsAPI бесплатного тарифа (100 запросов/день) — при большом портфеле и коротком интервале рассылки может не хватить, тогда автоматически используется Google News RSS.

## Возможные доработки

- Хранить историю цен в БД, чтобы не дёргать MOEX историю каждый раз.
- Добавить пороговые значения (% изменения) как настройку пользователя.
- Добавить уведомления о приближении даты погашения/оферты облигации (поле `MATDATE`/`OFFERDATE` в ISS).
- Многопользовательский rate-limit при рассылке (сейчас сообщения отправляются последовательно).
